# Google Sheet 寫入規則

## 寫入前提
只有在以下條件同時成立時才可寫入：
- 所有必填欄位已補齊
- 原提交者已作清楚確認

## V1 寫入策略
- 主資料寫入與圖片 upload 分開處理。
- 主資料一經成功寫入，即可回覆 `已成功儲存`。
- 圖片欄位由背景流程後補，不可阻塞主流程。

## 主資料寫入欄位
主資料寫入時至少包含：
- 門市編號
- 支票類別
- 輸入日期
- 支票付款日期
- 支票號碼
- 支票金額
- 收款公司名
- Invoice No.
- 備註
- 圖片狀態（預設 `待上傳`）
- 記錄鍵值

## 圖片回填欄位
背景 upload 成功後更新：
- 圖片連結
- 圖片預覽
- 圖片狀態（改為 `已上傳`）

背景 upload 失敗時更新：
- 圖片狀態（改為 `上傳失敗`）

## 更新識別原則
- 使用 `記錄鍵值` 更新既有列。
- 不可使用純 row number 作為唯一更新依據。

## 固定結果回覆
- 寫入成功：`已成功儲存`
- 寫入失敗：`不成功儲存`

## 寫入失敗後續
如寫入失敗，下一則訊息必須補充具體原因，例如：
- Apps Script 回傳錯誤
- token 驗證失敗
- sheet 名稱錯誤
- 未取得寫入設定

不可只停留在「寫入唔到」。

## 敏感設定原則
- Google Sheet 連結、Apps Script Web App URL、token、Google Drive folder id 等環境資料，不應硬編碼成可攜式 skill 的必要內容。
- 可攜式 skill 應保存流程與規則；實際 backend 設定應放在本地 workspace、部署文件或環境設定中。
