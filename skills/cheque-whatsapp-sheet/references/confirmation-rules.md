# 確認與修改規則

## 視為確認
以下回覆視為確認：
- `1`
- `確定`
- `OK`
- `好`

## 視為修改
以下回覆視為修改：
- `2`
- `更改`
- `修改`
- `有錯`

## 歧義處理
- 確認訊息應盡量直接 reply 助手上一則摘要。
- 如同時間有多張單，未 reply 對應摘要前，不可憑單獨一句 `1 / OK / 好` 直接寫入 Google Sheet。
- 如確認內容不明確，例如前面有兩張或以上待確認紀錄，而使用者只回 `ok`，應先要求對方指定是確認哪一張，不能自行判斷。
- 如使用者回覆 `2 / 更改 / 修改 / 有錯`，先不寫入表，應要求對方指出哪個欄位有錯，或補發更清晰圖片。

## 權限規則
- 只有原本提交該張單的使用者可以作最終 `1 / 確定 / OK / 好` 確認。
- 其他人即使回覆確認，也不可寫入 Google Sheet。
- 原提交者確認後，應立即執行寫入，不需再作第二次人手確認。
