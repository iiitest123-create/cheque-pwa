# 完整流程

## 輸入要求
最少必須包含：
- 門市編號
- 支票類別
- 相片

範例：
```text
PU003 鋪頭支票
```

## 主流程
1. 收到 WhatsApp 群組訊息。
2. 讀取支票／發票圖片。
3. 抽取欄位。
4. 用固定摘要格式回覆。
5. 等待原提交者確認。
6. 確認後先寫入 Google Sheet 主資料。
7. 將圖片處理交由背景流程，不可阻塞主流程。

## V1 圖片流程
1. 主資料寫入時，先將 `圖片狀態` 設為 `待上傳`。
2. 根據 session 記錄或媒體 mapping 找到對應 inbound 圖檔。
3. 背景上傳壓縮後圖片。
4. 取得圖片 URL 後，更新該列的 `圖片連結`、`圖片預覽`、`圖片狀態`。
5. 若圖片 upload 失敗，只更新狀態為 `上傳失敗`，不推翻主資料寫入結果。

## 固定摘要回覆格式
```text
- 門市編號：PU003
- 支票類別：鋪頭支票
- 輸入日期：2026-03-27
- 支票付款日期：2026-03-24
- 支票號碼：067175
- 支票金額：HK$715.30
- 收款公司名：有盛行辦館有限公司
- Invoice No.：526021362
- 備註：-

請回覆：
- 1 = 確定並寫入 Google Sheet
- 2 = 有錯，需修改
```

## 資料不足時回覆格式
```text
以下資料未能確認：
- 支票付款日期
- Invoice No.

請補拍清楚右上角日期及發票編號。
```

## 寫入結果固定字句
- 成功：`已成功儲存`
- 失敗：`不成功儲存`

## 圖片背景處理原則
- 不要等待圖片 upload 完成才回覆 `已成功儲存`。
- 不要在群組中為圖片 upload 成功額外洗版。
- 如背景 upload 失敗，優先在 Sheet 留下 `上傳失敗` 狀態供人工補處理。
