# 支票系統 PWA 第一期功能清單

## 目標
先把現有 WhatsApp 支票流程升級成一個可實際操作的 PWA 第一版，重點是：
- 店舖提交更方便
- 管理員確認更清楚
- 會計開始有對帳與提醒能力
- 保留現有 ledger / Google Sheet / 圖片留底能力

## 角色分工
### 1. 店舖版
只做最少輸入：
- 帳號綁定門市（預設只見自己門市）
- 新增記錄
- 選支票類別
- 拍照 / 上傳圖片
- 查看 OCR 抽取結果
- 手動更正欄位
- 提交等待確認
- 查看自己門市最近記錄

### 2. 管理員版
- 查看全部門市記錄
- 待確認箱
- 查看原圖 / 抽取結果
- 修改欄位
- 確認寫入 Google Sheet
- 查看寫入失敗原因
- 搜尋 / 篩選（門市、支票號碼、Invoice No.、公司名、日期）

### 3. 會計版
- 查看待過數支票
- 查看已到期未過數支票
- 標記已過數 / 未過數 / 異常
- 輸入對帳結果與跟進備註
- 查看提醒清單
- 日後接銀行資料匯入

## 第一期一定要有
### A. 基本提交
- 門市綁定登入
- 支票類別選單
- 圖片上傳（相機 / 相簿）
- OCR 結果預覽
- 可手動修正欄位
- 提交後產生 recordKey

### B. 管理確認
- 待確認清單
- 單筆詳情頁
- 原圖預覽
- 確認 / 修改 / 打回重交
- 寫入 Google Sheet
- 寫入結果與錯誤顯示

### C. 基本追蹤
- 記錄狀態：待確認 / 已寫入 / 寫入失敗 / 待過數 / 已過數 / 異常
- 圖片狀態：待上傳 / 已壓縮 / 已上傳 / 上傳失敗
- audit log：誰提交、誰確認、何時寫入

### D. 搜尋與查閱
- 搜尋 cheque no.
- 搜尋 invoice no.
- 搜尋公司名
- 按門市篩選
- 按日期範圍篩選

## 銀行對帳輸入層（已納入設計）
你剛剛補充的要求非常重要：
**銀行對帳輸入不只 CSV / Excel，亦要預留 PDF 或圖片。**

### 第一期先預留 3 種來源
1. **CSV / Excel**（最理想，日後最易自動 match）
2. **PDF 銀行月結單 / 流水單**
3. **圖片 / 截圖**（例如手機銀行畫面、銀行入數畫面）

### 輸入策略
- CSV / Excel：作為首選結構化資料來源
- PDF：先做文字抽取 / OCR，再進入對帳候選
- 圖片：走 OCR，再由人手確認 match

### 為何要三種都支援
因為實際環境未必每次都拎到乾淨 CSV；有時只有：
- PDF 月結單
- 銀行頁面截圖
- 手機 app 畫面圖片

如果系統只食 CSV，實戰就唔夠靈活。

## 對帳設計（第一期先做人手輔助）
### 對帳欄位
- 預計過數日
- 實際過數日
- 對帳狀態：未對帳 / 已對帳 / 有差異
- 支票狀態：待過數 / 已過數 / 異常
- 跟進備註

### 提醒規則
- 支票日前 2 日：列入「快到期」
- 支票日後 3 日未過數：列入「待跟進」
- 支票日後 7 日未過數：列入「異常」

## 第一期不急做
以下先不急，避免第一版太重：
- 原生 App Store / Play Store 上架
- 自動銀行 API 對接
- 全自動 PDF / 圖片對帳決策
- 多層審批流程
- 多公司 / 多帳套支援

## 建議開發順序
### Step 1
把現有 PWA 模板接上真資料：
- 店舖版提交
- 管理員待確認箱
- 單筆詳情頁

### Step 2
接現有 ledger / Google Sheet：
- 建立 record
- 寫表
- 顯示圖片與狀態

### Step 3
加會計版第一期：
- 手動標記過數
- 提醒清單
- 對帳備註

### Step 4
再加銀行對帳輸入層：
- CSV / Excel
- PDF
- 圖片

## 決策結論
PWA 第一版應先做到：
- **店舖提交順手**
- **管理員確認順手**
- **會計追支票順手**
- **銀行資料之後可由 CSV / PDF / 圖片輸入**
